# v3.2.5 — Changelog

Release date: 2026-04-28

---

## Bug Fixes

### Quotations PDF: Rupee symbol now renders correctly
The built-in Helvetica PDF font does not include the Indian Rupee sign (₹), causing it to render as "1" wherever it appeared — in the total row and in any item description containing the symbol. Switched the PDF font to Montserrat (embedded TTF), which fully supports the ₹ glyph. All text weights (regular, medium, semibold, bold, italic variants) are bundled locally.

### Quotations PDF: Footer no longer cramped with many items
When a quotation had enough items to fill most of the page, the footer section (bank details, totals, signature) was being compressed into the remaining space. Added `wrap={false}` to the footer so it stays intact — if items overflow the page, the footer moves to page 2 as a unit.

### Tasks: Comments no longer posted twice on slow connections
Clicking the submit button once could result in duplicate comments being saved to the database. This happened because the `sending` React state flag is asynchronous — a second call to `sendComment` (via keyboard shortcut or rapid clicks) could slip through before the first call had updated state. Fixed by adding a `useRef`-based synchronous lock (`sendingRef`) that blocks any concurrent invocation immediately.
