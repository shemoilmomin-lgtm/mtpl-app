## Bug fixes

### Quotation combobox showing legacy timestamp IDs
All 33 quotation records created before the MTPLQ naming convention was adopted had `q_<timestamp>` or `QT-xxxx` format IDs. The previous migration (v3.1.5) incorrectly set all records to `MTPLQ-xxxx`. This release corrects the DB with a two-prefix convention: id ≤ 41 → `QT-xxxx`, id > 41 → `MTPLQ-xxxx`.

The `QuotationCombobox` and the linked-quotation reference in the order view panel now derive the display label from `q.id` using this same logic instead of reading `q.quotation_id` directly from the database, ensuring the label is always correct regardless of what is stored in the `quotation_id` column.
