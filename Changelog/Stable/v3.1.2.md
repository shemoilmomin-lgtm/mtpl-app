# Stable v3.1.2 — Changelog

Release date: 2026-04-13

---

## Changes

### iPad layout optimisation
The entire app layout breakpoint was shifted from `md` (768px) to `lg` (1024px). iPad portrait (768–1023px) now uses the mobile layout — card lists, bottom navigation, and hamburger menu — instead of the cramped desktop table view. iPad landscape (1024px+) uses the desktop layout with sidebar and table.

On the Orders table (iPad landscape), the Proforma No. and Tax Invoice No. columns are hidden at `lg` and only appear at `xl` (1280px+), giving each column more breathing room at mid-width screens.

### App version bump
Version string in Settings updated to v3.1.2.

---

## Bug fixes

### "Check for Updates" button unresponsive
The button now shows "Checking…" while the service worker update check is in progress, then "Up to date" for 2.5 seconds if no update is found, or switches to "Update Now" if one is available. Previously the button appeared to do nothing.

### App update section visible on iPad portrait
The version / update section in Settings was hidden on iPad portrait due to `md:hidden`. Changed to `lg:hidden` to match the rest of the mobile layout boundary.
