# v3.1.1 — Changelog

Release date: 2026-04-13

---

## Bug fixes

### Proforma & Tax Invoice filters not working
The Proforma and Tax Invoice filters in the Orders list were checking incorrect field names (`proforma_number`, `tax_invoice_number`) that don't exist on order records. Fixed to use the correct fields (`proforma_invoice_number`, `invoice_number`). Both "Filled" and "Empty" options now work as expected.

### Orders list resets to page 1 after closing a record
After viewing or editing an order and closing the drawer, the list was always jumping back to page 1. The pagination state now persists — if you were on page 3, you return to page 3 after closing.

---

## Changes

### Orders filters panel redesign
The filter panel now uses a consistent 2-row grid layout (4 columns on desktop, 2 on mobile) instead of the previous flex-wrap approach that caused uneven sizing and large gaps. All controls are now equal width and uniformly spaced.
