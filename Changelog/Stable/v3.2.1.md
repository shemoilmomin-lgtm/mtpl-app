# v3.2.1 — Changelog

Release date: 2026-04-20

---

## Bug fixes

### Tasks: fix integer overflow error when opening a newly created task
When a task was created, the app temporarily assigned a negative timestamp-based ID for optimistic rendering. If the task detail panel (including the Comments tab) opened before the real DB ID was returned, it would fire a query with that temp ID, causing a PostgreSQL integer out-of-range error. The Comments tab now skips loading and SSE setup when the task ID is a temporary negative value.

### Tasks: fix comment submitted once appearing multiple times
After successfully posting a comment, the code was calling a full comment refetch (`load()`) in addition to the SSE live-update path. These two paths raced each other and could result in the same comment appearing two or three times. The refetch has been removed — the comment returned from the POST response is now inserted directly into state, and the SSE deduplication guard prevents any double-add.

### Orders: fix status dropdown not scrollable inside edit panel
The status selector dropdown in the orders edit panel had `max-h-80` applied but was missing `overflow-y-auto`, making it impossible to scroll when the list exceeded the available height. Scroll was also blocked by the modal's scroll-lock propagating through wheel events. Fixed by adding `overflow-y-auto` and stopping wheel event propagation on both the header and inline status selector dropdowns.
