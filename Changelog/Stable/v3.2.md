## Bug fixes

### Quotation PDF now shows correct quotation number
The downloadable PDF was displaying the raw internal `quotation_id` value (e.g. `q_<timestamp>`) instead of the formatted quotation number. Added `fmtQuotationNumber()` in `quotationPdf.jsx` that resolves the correct number to display:
- If `quotation_id` is already in `MTPLQ-xxxx` or `QT-xxxx` format, it is used as-is.
- Legacy records (created before the naming convention) fall back to deriving the number from the numeric `id`: IDs ≤ 41 render as `QT-xxxx`, IDs ≥ 42 render as `MTPLQ-xxxx`.

The downloaded PDF filename also uses the formatted number.
