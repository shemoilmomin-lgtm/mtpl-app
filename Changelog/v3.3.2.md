# v3.3.2 — Changelog

Release date: 2026-05-04

---

## Changes

### Mobile: Bottom Nav Bar Redesigned
The mobile bottom navigation bar height has been increased for easier tapping. Active tabs now display the icon inside a solid primary-colored pill with a white icon, making the active state clearly distinct. Label spacing is tightened for a more compact look.

### Profile Photos: Local Storage
Profile photos are no longer uploaded to cloud storage. They are now resized client-side to 350×350 and stored directly in the database as compressed JPEG data. This removes the dependency on R2 for avatars and ensures photos load instantly without expiring URLs. Uploading a new photo automatically replaces the old one.

### Fix: Invoice / Proforma / Delivery Edit Returning 403
The server route for inline-editing invoice number, proforma number, and delivery date was incorrectly restricted to admin and superadmin roles. The client-side had already been opened to all users in v3.3.1 but the server guard was not updated, causing a 403 on save for regular users. The server-side role check has been removed.

### Fix: Search Dialog Accessibility Warning
The global search command palette was rendering `DialogTitle` outside of `DialogContent`, causing a Radix accessibility warning on every page where it appeared. The title is now correctly placed inside `DialogContent` (visually hidden via `sr-only`).

### App Version in Settings
Version number is now sourced from `client/package.json` and injected at build time via Vite. The hardcoded `v3.3` strings in Settings have been replaced with the dynamic value. To update the version in future releases, only `client/package.json` needs to be changed.
