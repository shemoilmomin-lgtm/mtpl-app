## Changes

### Refresh button on mobile (Settings)
The "Check for Updates" button in the mobile Settings screen has been replaced with a simple "Refresh" button that reloads the app immediately.

---

## Bug fixes

### Global search now correctly opens the clicked record
Search results were navigating to the right page but not opening the specific record. Root cause: the search API returns all IDs as strings (`::text` cast), while loaded records in each page store integer IDs — the strict equality check always failed. Fixed by converting `item.id` to a `Number` before passing it in navigation state.
