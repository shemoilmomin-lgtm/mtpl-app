# Stable v3.1.4 — Changelog

Release date: 2026-04-13

---

## Bug fixes

### Notification badge not clearing after reading
Opening the notifications panel now immediately marks all notifications as read in the database. Previously the badge was only cleared in memory — on next page load it would reappear because the server still had them marked unread.

### Search dropdowns not scrollable on touch devices
Popover result lists across Orders, Quotations, Tasks, and Leads had scroll chaining issues on iPad and iPhone — touch scroll would bubble to the parent page instead of scrolling the list. Fixed by adding `overscroll-contain` to all popover list containers.

### Quotation combobox selected display
In the order edit form, when a quotation was selected the trigger showed the internal quotation ID first. Now shows a stacked layout: client name (top), quotation ID (middle), amount (bottom).

### Subtotal shown when no discount applied
In Quotations view and edit form, the Subtotal row now only appears when an actual discount is applied. When there is no discount, only the Total is shown.

---

## Changes

### Dashboard: Recent Notifications replaces Recent Orders
The "Recent Orders" card on the Dashboard has been replaced with a "Notifications" card showing the 6 most recent notifications with type icon, message, relative timestamp, and an unread indicator dot.
